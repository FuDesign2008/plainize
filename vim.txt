<p><b>Vim</b> (<span class="nowrap"><span title="Representation in the International Phonetic Alphabet (IPA)" class="IPA"><a href="/wiki/Help:IPA_for_English" title="Help:IPA for English">/</a></span><span class="IPA nopopups"><a href="/wiki/Help:IPA_for_English#Key" title="Help:IPA for English"><span title="'v' in 'vie'" style="border-bottom:1px dotted">v</span></a></span><span class="IPA nopopups"><a href="/wiki/Help:IPA_for_English#Key" title="Help:IPA for English"><span title="/ɪ/ short 'i' in 'bid'" style="border-bottom:1px dotted">ɪ</span></a></span><span class="IPA nopopups"><a href="/wiki/Help:IPA_for_English#Key" title="Help:IPA for English"><span title="'m' in 'my'" style="border-bottom:1px dotted">m</span></a></span><span title="Representation in the International Phonetic Alphabet (IPA)" class="IPA"><a href="/wiki/Help:IPA_for_English" title="Help:IPA for English">/</a></span></span>;<sup id="cite_ref-pronounc_3-0" class="reference"><a href="#cite_note-pronounc-3"><span>[</span>3<span>]</span></a></sup> a contraction of <b>Vi IMproved</b>) is a <a href="/wiki/Text_editor" title="Text editor">text editor</a> written by <a href="/wiki/Bram_Moolenaar" title="Bram Moolenaar">Bram Moolenaar</a> and first released publicly in 1991. Based on the ideas of the <a href="/wiki/Vi" title="Vi">vi</a> editor common to <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems, Vim is designed for use both from a <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a> and as a standalone application in a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a>. Vim is <a href="/wiki/Free_and_open_source_software" title="Free and open source software" class="mw-redirect">free and open source software</a> and is released under a license that includes some <a href="/wiki/Charityware" title="Charityware" class="mw-redirect">charityware</a> clauses, encouraging users who enjoy the software to consider donating to children in <a href="/wiki/Uganda" title="Uganda">Uganda</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup> The license is compatible with the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>.</p>
<p>Although Vim was originally released for the <a href="/wiki/Amiga" title="Amiga">Amiga</a>, Vim has since been developed to be <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a>, supporting <a href="#Availability">many other platforms</a>. In 2006, it was voted the most popular editor amongst <a href="/wiki/Linux_Journal" title="Linux Journal">Linux Journal</a> readers.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span>[</span>5<span>]</span></a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vim_(text_editor)&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Bram_Moolenaar" title="Bram Moolenaar">Bram Moolenaar</a> began working on Vim for the <a href="/wiki/Amiga" title="Amiga">Amiga</a> computer in 1988. Moolenaar first publicly released Vim (v1.14) in 1991.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span>[</span>6<span>]</span></a></sup> Vim was based on an earlier editor, Stevie, for the Atari ST,<sup id="cite_ref-Ryan_1-1" class="reference"><a href="#cite_note-Ryan-1"><span>[</span>1<span>]</span></a></sup> created by Tim Thompson, Tony Andrews and G.R. (Fred) Walter.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2011)">citation needed</span></a></i>]</sup></p>
<p>The name "Vim" is an acronym for "<i>Vi IMproved</i>"<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span>[</span>7<span>]</span></a></sup> because Vim is an extended version of the <a href="/wiki/Vi" title="Vi">vi</a> editor, with many additional features designed to be helpful in editing program <a href="/wiki/Source_code" title="Source code">source code</a>. Originally, the acronym stood for "<i>Vi IMitation</i>", but that was changed with the release of Vim 2.0 in December 1993.<sup id="cite_ref-filewatcher_8-0" class="reference"><a href="#cite_note-filewatcher-8"><span>[</span>8<span>]</span></a></sup> A later comment states that the reason for changing the name was that Vim's feature set surpassed that of vi.<sup id="cite_ref-history_9-0" class="reference"><a href="#cite_note-history-9"><span>[</span>9<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Interface">Interface</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vim_(text_editor)&amp;action=edit&amp;section=2" title="Edit section: Interface">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:202px;"><a href="/wiki/File:Vim.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/28/Vim.png/200px-Vim.png" width="200" height="208" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/28/Vim.png/300px-Vim.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/28/Vim.png/400px-Vim.png 2x" data-file-width="662" data-file-height="688"></a>
<div class="thumbcaption">
<div class="magnify"><a href="/wiki/File:Vim.png" class="internal" title="Enlarge"></a></div>
Graphical Vim under <a href="/wiki/GTK%2B" title="GTK+">GTK+ 2</a>.</div>

<p>Like <a href="/wiki/Vi" title="Vi">vi</a>, Vim's interface is not based on menus or icons but on commands given in a <a href="/wiki/Text_user_interface" title="Text user interface" class="mw-redirect">text user interface</a>; its <a href="/wiki/GUI" title="GUI" class="mw-redirect">GUI</a> mode, <b>gVim</b>, adds menus and toolbars for commonly used commands but the full functionality is still expressed through its <a href="/wiki/Command_line" title="Command line" class="mw-redirect">command line</a> mode. Vi (and by extension Vim) tends to allow a typist to keep their fingers on the home row, which can be an advantage for a <a href="/wiki/Touch_typist" title="Touch typist" class="mw-redirect">touch typist</a>.<sup id="cite_ref-Lamb1998_10-0" class="reference"><a href="#cite_note-Lamb1998-10"><span>[</span>10<span>]</span></a></sup></p>
<p>Vim has a built-in <a href="/wiki/Tutorial#Computer_based_tutoring" title="Tutorial">tutorial</a> for beginners (accessible through the "vimtutor" command). There is also the Vim <a href="/wiki/User_guide" title="User guide">Users' Manual</a> that details Vim's features. This manual can be read from within Vim, or found online.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span>[</span>11<span>]</span></a></sup><sup id="cite_ref-vimbook_12-0" class="reference"><a href="#cite_note-vimbook-12"><span>[</span>12<span>]</span></a></sup></p>
<p>Vim also has a built-in help facility (using the <code>:help</code> command) that allows users to query and navigate through commands and features.</p>
<h2><span class="mw-headline" id="Customization">Customization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vim_(text_editor)&amp;action=edit&amp;section=3" title="Edit section: Customization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Part of Vim's power is that it can be extensively customized. The basic interface can be controlled by the many options available, and the user can define personalized key mappings—often called <a href="/wiki/Macro_(computer_science)#Keyboard_macros" title="Macro (computer science)">macros</a>—or abbreviations to automate sequences of keystrokes, or even call internal or user defined functions.</p>
<p>There are many <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugins</a> available that will extend or add new functionality to Vim. These complex scripts are usually written in Vim's internal scripting language <i><a href="/wiki/Vimscript" title="Vimscript" class="mw-redirect">vimscript</a></i>.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span>[</span>13<span>]</span></a></sup> Vim also supports scripting using <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> (as of Vim 7.3), <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Racket_(programming_language)" title="Racket (programming language)">Racket</a><sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span>[</span>14<span>]</span></a></sup> (formerly PLT Scheme), <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, and <a href="/wiki/Tcl" title="Tcl">Tcl</a>.</p>
<p>There are projects bundling together complex scripts and customizations and aimed at turning Vim into a tool for a specific task or adding a major flavour to its behaviour. Examples include <a href="/wiki/Cream_(software)" title="Cream (software)">Cream</a>, which makes Vim behave like a click-and-type editor, or VimOutliner, which provides a comfortable <a href="/wiki/Outliner" title="Outliner">outliner</a> for users of Unix-like systems.</p>
<h2><span class="mw-headline" id="Features_and_improvements_over_vi">Features and improvements over vi</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vim_(text_editor)&amp;action=edit&amp;section=4" title="Edit section: Features and improvements over vi">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Vim has a vi compatibility mode but when not in this mode Vim has many enhancements over vi.<sup id="cite_ref-vimhelp_15-0" class="reference"><a href="#cite_note-vimhelp-15"><span>[</span>15<span>]</span></a></sup> However, even in compatibility mode, Vim is not 100% compatible with vi as defined in the <a href="/wiki/Single_Unix_Specification" title="Single Unix Specification" class="mw-redirect">Single Unix Specification</a><sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span>[</span>16<span>]</span></a></sup> and <a href="/wiki/POSIX" title="POSIX">POSIX</a> (e.g., Vim does not support vi's open mode, only visual mode). Vim has nevertheless been described as “very much compatible with Vi”.<sup id="cite_ref-vimfaq_17-0" class="reference"><a href="#cite_note-vimfaq-17"><span>[</span>17<span>]</span></a></sup></p>
<p>Some of Vim's enhancements include <a href="/wiki/Autocomplete" title="Autocomplete">completion</a>, <a href="/wiki/Data_comparison" title="Data comparison">comparison</a> and <a href="/wiki/Merge_(revision_control)" title="Merge (revision control)">merging</a> of files (known as vimdiff), a comprehensive integrated help system, extended <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>, <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a> (both native and through alternative scripting interpreters such as Perl, Python, Ruby, Tcl, etc.) including support for <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugins</a>, a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> (known as gvim), limited <a href="/wiki/Integrated_development_environment" title="Integrated development environment">integrated development environment</a>-like features, <a href="/wiki/Mouse_(computing)" title="Mouse (computing)">mouse</a> interaction (both with and without the GUI), <a href="/wiki/Folding_editor" title="Folding editor">folding</a>, editing of compressed or archived files in <a href="/wiki/Gzip" title="Gzip">gzip</a>, <a href="/wiki/Bzip2" title="Bzip2">bzip2</a>, <a href="/wiki/ZIP_(file_format)" title="ZIP (file format)" class="mw-redirect">zip</a>, and <a href="/wiki/Tar_(file_format)" title="Tar (file format)" class="mw-redirect">tar</a> format and files over network protocols such as <a href="/wiki/Secure_Shell" title="Secure Shell">SSH</a>, <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a>, and <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">HTTP</a>, session state preservation, <a href="/wiki/Spell_checker" title="Spell checker">spell checking</a>, split (horizontal and vertical) and tabbed windows, <a href="/wiki/Unicode" title="Unicode">unicode</a> and other multi-language support, <a href="/wiki/Syntax_highlighting" title="Syntax highlighting">syntax highlighting</a>, trans-session command, search and cursor position <a href="/wiki/Command_history" title="Command history">histories</a>, multiple level and branching <a href="/wiki/Undo" title="Undo">undo/redo</a> history which can persist across editing sessions, and visual mode.</p>
<h2><span class="mw-headline" id="Availability">Availability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vim_(text_editor)&amp;action=edit&amp;section=5" title="Edit section: Availability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Whereas <i>vi</i> was originally available only on <a href="/wiki/Unix" title="Unix">Unix</a> operating systems, Vim has been <a href="/wiki/Porting" title="Porting">ported</a> to many operating systems including <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a> (the initial target platform), <a href="/wiki/Atari" title="Atari">Atari</a> <a href="/wiki/MiNT" title="MiNT">MiNT</a>, <a href="/wiki/BeOS" title="BeOS">BeOS</a>, <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> starting from <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/OS/390" title="OS/390">OS/390</a>, <a href="/wiki/MorphOS" title="MorphOS">MorphOS</a>, <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, <a href="/wiki/QNX" title="QNX">QNX</a>, <a href="/wiki/RISC_OS" title="RISC OS">RISC OS</a>, <a href="/wiki/GNU/Linux" title="GNU/Linux" class="mw-redirect">GNU/Linux</a>, <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD</a>, and <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS" class="mw-redirect">Classic Mac OS</a>.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18"><span>[</span>18<span>]</span></a></sup> Also, Vim is shipped with every copy of <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> <a href="/wiki/OS_X" title="OS X">OS X</a>.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19"><span>[</span>19<span>]</span></a></sup></p>
<p>Independent ports of Vim are available both for <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a><sup id="cite_ref-vimtouch_20-0" class="reference"><a href="#cite_note-vimtouch-20"><span>[</span>20<span>]</span></a></sup> and <a href="/wiki/IOS" title="IOS">iOS</a>.<sup id="cite_ref-vimiosappstore_21-0" class="reference"><a href="#cite_note-vimiosappstore-21"><span>[</span>21<span>]</span></a></sup></p>
